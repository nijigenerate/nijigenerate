# 05. ステップ 2：首・顔・目・口の親子関係を整える

次に、首から上の構造を整えます。  
ここを正しく作っておくと、首を回したときに顔全体が自然に付いてくるようになります。

## 理想的な関係

首、顔、目、口は、見た目の親子関係に合わせて配置します。

- 首（`Neck` の Part）は胴（`Body` の Part）の子にする
- 顔（`Face` の Part）は首（`Neck` の Part）の子にする
- 右目、左目、口、耳の各 Part は顔（`Face` の Part）の子にする
- 目と口は、必要に応じて `DynamicComposite` でまとめる
- 影用の Part は、影を落とされる本体 Part の子にする

ここで重要なのは、親が `Part` であることです。`GridDeformer` や `Node` の下に入っているだけでは、Part 間の関係としては不十分な場合があります。

## よくある失敗

- 顔の Part を胴体に直接ぶら下げてしまう
- `GridDeformer` や `Node` の下に入れたことで、顔や首の親子関係も正しくなったと誤解する
- 目や口の影を、本体 Part とは別の場所に置いてしまう

親子関係がずれていると、首や顔を動かしたときに、目、口、影が置いていかれることがあります。

## 操作例

1. 胴体の Part、または胴体側の適切な親の下に `Neck` Part を配置する。
2. `Neck` の子として `Face` Part を配置する。
3. 顔の中の Part、つまり目、口、耳などを `Face` の子にまとめる。
4. 目や口の差分が複数ある場合は、`DynamicComposite` でまとめる。
5. 影用の Part は、本体 Part の子になるように移動する。
