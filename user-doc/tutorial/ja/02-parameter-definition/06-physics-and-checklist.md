# 06. 物理系とチェックポイント

髪・服・リボンなどの揺れは、顔や口の表情パラメータとは分けて考えます。  
最初からすべてを作り込む必要はありませんが、どの方式で扱うかは決めておきます。

## PathDeformer で髪を扱う場合

髪の束を `PathDeformer` で扱う場合は、`PathDeformer` の物理を有効化します。

**向いているケース**

- 髪の束が細長く、付け根から先端へしなる
- 変形の流れをパスで制御したい
- 髪の先端が揺れる表現を作りたい

この場合、通常の表情パラメータとして `Hair::Physics` を無理に作るより、`PathDeformer` 側の物理を使う方が自然です。

## SimplePhysics で髪や服を扱う場合

髪や服を通常の `SimplePhysics` で扱う場合は、対象 Part に `SimplePhysics` ノードを追加します。  
そのうえで、必要なら 2 次元パラメータを作ります。

**名前の例**

- `HairFront::Physics`
- `HairSide::Physics`
- `Ribbon::Physics`
- `Sleeve::Physics`

`<パーツ名>::Physics` のようにすると、どの物理パラメータがどの Part に対応するか分かりやすくなります。

## 判断基準

- 付け根から先端へしなる長い髪束は `PathDeformer`
- 小さな揺れ物や、Part 単位で揺らしたい髪・服・リボンは `SimplePhysics`
- 表情に連動して意図的に形を変えたいものは通常パラメータ
- 慣性や揺れとして自然に遅れて動かしたいものは物理

## 最後にチェックしたいポイント

基本パラメータを作ったら、次の点を確認します。

- `Face::Yaw-Pitch` は `(-1,-1)` から `(1,1)`、デフォルト `(0,0)` か
- `Face::Roll` は `-1..1`、デフォルト `0` か
- `Body::Yaw-Pitch` は `(-1,-1)` から `(1,1)`、デフォルト `(0,0)` か
- `Body::Roll` は `-1..1`、デフォルト `0` か
- `Eye::L::Blink` / `Eye::R::Blink` は X が `0..1`、Y が `-1..1` か
- `Eye::L::X-Y` / `Eye::R::X-Y` は `(-1,-1)` から `(1,1)` か
- `Eyebrow::L` / `Eyebrow::R` は `(-1,-1)` から `(1,1)` か
- `Mouth::Open` は X が `0..1`、Y が `-1..1` か
- 顔・胴体・目・眉・口のパラメータを同時に動かしても破綻していないか
- 物理系パラメータが、顔や口の表情パラメータと混ざっていないか

## ここまでの到達点

ここまでできていれば、表情と向きの基本入力が整理された状態になります。  
このあと細かい表情差分や髪・服の物理を作り込む場合も、基本パラメータと物理パラメータが分かれているため、調整しやすくなります。
